SQL Skills Used:

  • Data Retrieval (SELECT statements): Extracting specific columns and rows based on conditions.
  • Filtering Data (WHERE clause): Using conditional logic to retrieve relevant records (e.g., by city, salary range, manager ID).
  • Joining Tables: LEFT JOIN: To combine data from Employee_Details and Employee_Salary based on matching EmpID.
    FULL JOIN: To capture all records from both tables, including unmatched entries.
  • Aggregation Functions: COUNT(): For counting records (e.g., number of employees in a project).
    MAX(), MIN(), AVG(): For calculating salary statistics.
  • Grouping Data (GROUP BY): Summarizing data by employee to compute total compensation.
  • Pattern Matching (LIKE operator): Using wildcards to find names with specific character patterns.
  • Set-Based Operations: Applying conditions like NOT IN to exclude specific project types.
  • Mathematical Operations: Performing arithmetic (e.g., adding base salary and variable pay).
  • Range Filtering (BETWEEN): Extracting salary values within a defined range.
  • Use of DISTINCT (if implied in unique project extraction): Returning unique project names.

Project Description

This project centers on the analysis of employee data stored across two interconnected tables(Employee_Details and Employee_Salary) with the objective of deriving valuable operational and HR insights using SQL. Through structured queries, the project demonstrates key database concepts such as filtering, joining, aggregation, and pattern matching to support strategic decision-making. The project objectives includes:

Data Exploration & Understanding:

Analyze employee demographic, and salary data using structured SQL queries.

Operational Insight Generation:

Provide clarity on team structures, salary trends, and project distributions to inform business decisions.

Cross-Table Analysis:

Integrate data from multiple tables to support comprehensive reporting on employee status and compensation.

Key Analytical Use Cases and Insights Delivered:

  • Managerial Oversight: Identified employees reporting to specific managers.
  • Project Allocation Overview: Extracted unique project codes to understand resource distribution.
  • Team Size Analysis: Counted employees assigned to key projects such as ‘P1’.
  • Compensation Review: Calculated minimum, maximum, and average salary across the organization.
  • Targeted Pay Band Reporting: Isolated employees earning within a specified salary range (£9,000–£15,000).
  • Location-Based Filtering: Retrieved employees based on combined geographic and managerial criteria.
  • Flexible Query Logic: Applied compound filtering using OR to support broader employee searches.
  • Exclusion Queries: Listed employees involved in all projects except ‘P1’.
  • Total Earnings Calculation: Computed complete salary packages by summing base salary with variable components.
  • Name Pattern Matching: Found employees using advanced search logic with wildcards (e.g., names containing “hn”).
  • Data Merging & Reconciliation: Used joins to assess completeness and overlap between employee records across tables.

Impact of the Project

By completing this SQL project, a fully operational query-based analysis framework was developed to enable rapid, flexible querying of employee data. The insights generated support stakeholders (including HR managers, project leads, and finance teams) in making data-informed decisions regarding team composition, project staffing, salary distribution, and workforce planning.
This project showcases the practical application of SQL for real-world business analysis and underlines the importance of relational data modeling, query optimization, and analytical thinking in a corporate context.
