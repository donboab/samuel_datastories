SQL Skills Used:

  • Data Aggregation: These was used for summarizing incident. Example like: COUNT(), SUM(), AVG(), ROUND(); counts, total costs, and average values across categories
  • Grouping and Categorization: GROUP BY was used for breaking down incidents by department, plant, shift, and more
  • Filtering Data: WHERE clauses was used to isolate specific incident types, report types, or date ranges. Example: Filtering for "Lost Time", "Vehicle" incidents, or days lost > 2
  • Sorting and Ranking: ORDER BY was used with DESC/ASC to prioritize high-impact areas (e.g., top departments or incident types)
  • Limiting Results: TOP (n) was used to retrieve highest or lowest ranking items (e.g., most affected age group, top 5 plants)
  • Date and Time Functions: YEAR(), MONTH(), FORMAT(), DATEPART() were used to extract and organize data by time dimensions (e.g., incidents by month or year)
  • Common Table Expressions (CTEs): Was used to simplify complex queries, such as month name extraction for ranking incident frequency
  • Descriptive Analytics: Writing diagnostic and exploratory queries to uncover trends, patterns, and root causes
  • Data Validation & Cleaning: Ensuring correct data types and consistent formatting for reliable analysis (implicit skill in query design)

Project Description

This project involved the exploration and analysis of a workplace safety dataset comprising detailed incident records across multiple operational plants. The goal was to leverage SQL to extract actionable insights that inform health and safety decision-making, operational improvements, and risk mitigation strategies. The analytical focus areas are:

Cost Impact Assessment:

  • Yearly trends in safety-related expenditure
  • Identification of high-cost incident types (e.g., 'Vehicle' incidents)
  • Total and average incident costs by injury location, report type, and department

Severity and Duration Metrics:

  • Days lost per incident type and department
  • Average days lost by incident category
  • Filtering departments with recurring incidents exceeding 2-day loss

Technology used:

SQL Server Management Studio (SSMS) for querying and analysis
